# 🐧 在 Linux 中不落地执行远程脚本

在某些自动化场景中，我们希望 **不将脚本写入磁盘**，而是直接从 HTTP 加载并在内存中执行。这在本地服务（如 `127.0.0.1`）、CI/CD、内网自动化等环境中非常常见。

下面是常用的几种“不落地执行”方式。

---

## 📌 直接通过 `curl` 执行

```bash
curl -s http://127.0.0.1/1.sh | bash
```

流经管道的数据直接传给 `bash`，不会保存到硬盘。

---

## 📌 使用 `wget` 执行

```bash
wget -qO- http://127.0.0.1/1.sh | bash
```

`-qO-` 表示静默模式并将内容输出到 stdout，再传给 `bash`。

---

## 📌 指定解释器执行（例如 sh）

```bash
curl -s http://127.0.0.1/1.sh | sh
```

适用于没有 shebang 或兼容性需求的情况。

---

## 📌 传递参数给远程脚本

```bash
curl -s http://127.0.0.1/1.sh | bash -s -- arg1 arg2
```

所有参数会透传给脚本。

---

## 📌 不落地执行其他语言脚本

例如 Python：

```bash
curl -s http://127.0.0.1/1.py | python3
```

同样不会写入任意文件。

---

## ⚠️ 安全注意事项

虽然本地地址（如 `127.0.0.1`）通常安全，但直接将脚本通过管道喂给 shell 在一般场景下风险极高。

仅在以下情况下使用：

- 内网环境  
- 自己管理的服务器  
- CI/CD pipeline  
- 你完全信任脚本来源  

---

## 🧠 工作原理（简述）

Shell 在接到管道输入后，会直接从 **标准输入（stdin）** 执行脚本内容。整个过程只在内存中进行，因此被称为“不落地执行”。
